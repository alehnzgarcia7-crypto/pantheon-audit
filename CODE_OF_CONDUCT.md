# Code of Conduct

PANTHEON is a security research tool. Contributors and users behave with the professionalism appropriate to security research.

## Expected behavior
- Audit only systems you own or have explicit written authorization to audit.
- Respect responsible disclosure timelines for vulnerabilities discovered using PANTHEON.
- Do not use PANTHEON to facilitate unauthorized access, fraud, or illegal activity.
- When reporting bugs in PANTHEON, do not include sensitive customer data from your audit subjects.
- When contributing personas or framework mappings, cite public sources accurately.

## Unacceptable behavior
- Using PANTHEON against systems you do not own or have permission to test.
- Publishing audit reports of third parties without their consent.
- Misrepresenting PANTHEON output as the result of a human-led tier-S audit.
- Stripping the adversarial gate to inflate finding counts.
- Adding personas, frameworks, or templates that misrepresent the methodologies they claim to follow.

## Reporting violations
Contact the maintainer (@nexogeopo) directly. Investigations are confidential.

## Scope
This code applies to all PANTHEON contributions, discussions, and uses of the plugin name.
