# CNBV CUIFPEs 2026 Vendor Authorization

Critical for Phoenix Financial OS and any IFPE deploying third-party services.

## Sanctions matrix (2026)
- Base fine: 30,000 to 150,000 UMAs (MXN $3.5M to $17.6M)
- Operations suspension: temporary or permanent per Articles 68 and 69 of Ley Fintech
- License revocation
- Personal liability for IFPE General Director per Article 44 CUIFPEs
- LFPDPPP 2025 additive sanctions when personal data involved: +320,000 UMAs

## Filing-to-resolution timeline
6 to 8 months typical with multiple oficio de observaciones cycles. Plan accordingly.

## Vendor categories requiring CNBV authorization
- Critical infrastructure: cloud hosting (per LATAM cloud-localization rules), core banking platforms, payment processors
- Authentication providers: KYC vendors (MetaMap, Truora), biometric services
- Data processors: analytics SDKs, marketing platforms with PII access
- ID-verification services
- Open Finance API providers (when Article 76 implementation lands)

## Audit checkpoint
For each critical vendor, verify ONE of:
- (a) CNBV authorization on file with current vigencia
- (b) Notice filed with CNBV plus acknowledgement
- (c) Justification documented as non-critical (rare)

Absence is Critical. Document in deliverable `cnbv-vendor-authorization-matrix.md`.
