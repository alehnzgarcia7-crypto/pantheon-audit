# PANTHEON-AUDIT REPO MEMORY

## Repo purpose
This repo is `pantheon-audit`, a Claude Code plugin implementing a high-assurance security audit workflow with eight auditor personas, twelve phases, adversarial gate verification, compliance auto-mapping to eighteen frameworks, and thirty-five auto-generated deliverables.

## Maintainer
Alex (@nexogeopo). Operates in Spanish but maintains source in English.

## Style rules
1. **No em-dashes anywhere.** Use commas, parentheses, periods, or restructure.
2. **No marketing language.** Avoid: revolutionary, groundbreaking, game-changing, paradigm-shifting, breakthrough, world-class, cutting-edge.
3. **No emoji in source files.** Exception: README.md shields.io badges.
4. **English in source, user-configurable for runtime output.**
5. **Trail of Bits style for findings.** Strict adherence to PANTHEON-NNNN format.
6. **Opus 4.7 max in all sub-agents.** No Sonnet. No reduced effort.

## Development principles
- Sub-agent system prompts are contracts. Add specificity, do not remove constraints.
- Adversarial gate is non-negotiable. No bypass mechanisms.
- Compliance mappings are evidence-driven.
- State schema is versioned. Breaking changes require schema_version bump.

## Common tasks
- **Add persona**: `plugins/pantheon/agents/<name>.md` + update `references/persona-doctrines.md` + orchestrator skill + `references/12-phases-spec.md`.
- **Add framework**: `plugins/pantheon/skills/pantheon-compliance-mapper/references/18-frameworks-catalog.md` + mapper SKILL.md + finding format spec.
- **Add deliverable**: `plugins/pantheon/skills/pantheon-deliverables-generator/templates/NN-name.md` + orchestrator final logic + deliverable catalog.
